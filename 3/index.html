Overview

This project is divided into two parts. Part A explores manual image warping and mosaicing techniques where I implemented homography computation, image warping with multiple interpolation methods, and blended multiple images together to create seamless panoramic mosaics. Part B extends this work by implementing an automatic feature detection and matching system that eliminates the need for manual point correspondence selection, using Harris corner detection, feature descriptors, and RANSAC for robust homography estimation.

Mathematical Formulation

For each correspondence point pair (x, y) → (x', y'), I set up two equations in the homography matrix H. This creates a system of equations Ah = b, where h is a vector containing the 8 unknown entries of H. With n point correspondences, I get 2n equations. I solve this overdetermined system using least squares (np.linalg.lstsq) to find the homography that best fits all correspondences, making it robust to small errors in point selection.

The equations are structured as follows:

  • For each point: A contains the coefficients based on (x, y) coordinates. For each (x, y) and (x', y'), there is a row of A that looks like: [x y 1 0 0 0 -x(x') -y(x')] and another row that looks like [0 0 0 x y 1 -x(y') -y(y')]
  • For each point, b will contain a row with the value [x'] and a row with the value [y']
  • H = [h_1 h_2 h_3 h_4 h_5 h_6 h_7 h_8], where H = [[h_1 h_2 h_3] [h_4 h_5 h_6] [h_7 h_8 1]]
  • Solving Ah = b gives us the 8 parameters of the homography matrix

Interpolation Comparison

Quality: The quality of the resulting image is better with bilinear interpolation than with the nearest neighbor approach. This is because the nearest neighbor approach tries to map partial pixels to the nearest full pixel, meaning that we lose the nuance and detail that may have been spread across pixels. For example, it is possible that some pixels in the original are never corresponding to a rounded pixel in the new version, meaning that that pixel value is completely lost. However, with the bilinear, the pixels will still contribute to the result, even if their weight is not 100%.

Speed: The nearest neighbor approach is quicker (7.0736 seconds instead of 7.3857 seconds after computing the homography). This is due to the simpler logic required of just rounding instead of checking more conditions. However, this is not a significant difference in time for the smaller imagegs that are lower quality and when we are only processing one image. If we have to process more images on a larger scale, we would have to take this difference into account more, as it would be compounded.

Part A.4: Blend the Images into a Mosaic

Blending Procedure

My blending approach involves an alpha channel and laplacian pyramid. For each image, I create an alpha channel, which is the same dimensions as the output result, and has value 1 where the pixels are filled in using values from the original, and 0 everywhere else. I then use distance_transform_edt (the scipy version of bwdist) to create a dropoff on the weighting, where the closer to the center of the original image, the more it is weighted. I then determine the location of the overlap region (where both alpha channels equal one). For that area, I normalize the weight, so that the overall weight is equal to 1 for each pixel across the two images. I do this by dividing the weight of each image by the overall weight in that region. Where the total distance is 0, such as at the edge of both of the images, I will weight them evenly, with 0.5 each. I then get a laplacian stack. For the lower frequencies, I will weight the level from the laplacian stack by the alpha channel that I just created. For the highest frequency, I create a binary alpha channel, where the channel is 1 if the values in the channel were greater than 0, and 0 otherwise. This allows for the edges to be more clearly maintained, while the colors in the lower frequencies can be better blended more smoothly. I then collapse the modified and weighted laplacian.

Part B: Feature Matching for Autostitching

Part B.1: Harris Corner Detection

Harris Interest Points

I implemented Harris corner detection to identify interest points in images. The Harris detector finds corners by analyzing the intensity gradient in different directions. Initially, this produces many corner detections, including many redundant points clustered in the same regions.

Adaptive Non-Maximal Suppression (ANMS)

To select a well-distributed subset of the strongest corners, I implemented Adaptive Non-Maximal Suppression. ANMS ensures that selected corners are spatially distributed across the image rather than clustered in high-contrast regions. Each corner is only kept if it is the strongest corner within a certain radius, where that radius is as large as possible. This gives us a more evenly distributed set of feature points suitable for matching.

Part B.2: Feature Descriptor Extraction

Implementation

I implemented feature descriptor extraction by sampling 8x8 patches from larger 40x40 windows around each detected corner. The larger window is first blurred with a Gaussian filter to create a more robust descriptor that is less sensitive to small pixel-level variations. The 8x8 patch is then extracted by downsampling from this blurred 40x40 region. Each descriptor is bias/gain normalized by subtracting the mean and dividing by the standard deviation, which makes the descriptors more robust to illumination changes.
